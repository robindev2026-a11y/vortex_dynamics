# Vortiq Dynamics Website Plan

## Objective

Create a static lead-generation website for Vortiq Dynamics, a technology solutions company building IoT, cloud, web, mobile, embedded systems, and edge AI solutions for startups, businesses, and industries.

The site should feel exciting, engineering-led, trustworthy, modern, and conversion-focused. It should borrow the confidence, scale, and mission-driven energy of a serious advanced-technology company while staying practical for business buyers who want to request a quote.

## Primary Audience

- Startup founders who need an MVP, prototype, or full product build
- Business owners who need websites, dashboards, portals, and automation
- Industrial and operations teams looking for IoT monitoring or connected systems
- Companies needing embedded systems, firmware, hardware integration, or edge AI
- Hiring candidates interested in joining Vortiq Dynamics

## Core User Goals

- Understand what Vortiq Dynamics builds
- See whether the company can handle both software and hardware work
- Explore services and technical capabilities
- Review credible example case studies
- Request a quote for a project
- Contact the team
- Apply for career opportunities

## Media and Image Strategy

Initial version:
- Use polished placeholders for hero, services, case studies, gallery/work images, and about sections.
- Placeholders should clearly represent technology work: PCB boards, embedded devices, dashboards, cloud systems, mobile UI, engineering work, and IoT monitoring.

Future version:
- Replace placeholders with Vortiq Dynamics real work images.
- Each major section should be able to accept its own image from a future admin panel.
- Images should support an ambient background treatment where the image appears clearly in front and a blurred, enlarged copy of the same image creates a soft contrast-matching background.

Ambient image treatment:
- Main image is sharp and readable.
- Background copy is enlarged, blurred, and softened behind the main image.
- The effect should feel like the background picks up the image's color mood without using a hard frame.
- Advanced color extraction can be added later, but the first version should use the lightweight duplicated-image method.

## Future Admin-Ready Content Structure

The admin panel is not part of the first website build. The website should still be prepared for it by organizing content as structured data.

Recommended content sources:
- Static JSON or TypeScript data files for the first version
- Later CMS/API response from the admin panel

Admin-editable content groups:
- Header navigation
- Hero headline, subtext, CTAs, and media
- About section text and image
- Service cards
- Capability flow items
- Tech stack categories
- Case studies
- Project/gallery images
- FAQ items
- Careers content
- Contact details
- Footer links and social links

## Recommended Static Build Approach

Good options:
- Plain HTML, CSS, and JavaScript for a simple static site
- Astro for a fast, content-focused static site
- Next.js static export if the project may grow into a larger app

Recommended choice:
Astro is the best fit if admin-ready structured content is important. It keeps the first site fast and static while making it easier to render pages from JSON or future CMS/API data.
