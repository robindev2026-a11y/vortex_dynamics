# Static Website Creation Prompt

Use this prompt to generate or build the Vortiq Dynamics static website.

```md
Create a bold, modern, static website for Vortiq Dynamics, a technology solutions company that builds IoT, cloud, web, mobile, embedded systems, and edge AI solutions for startups, businesses, and industries.

The website should feel exciting, trustworthy, engineering-led, and premium. Take inspiration from the confident, mission-driven energy of advanced technology companies like Shield AI, but do not copy their design. Translate that feeling into a clear business website for a technology services company.

Build the following pages:
- Home
- Services
- Solutions
- Case Studies
- About
- Careers
- Contact
- Request Quote

Use this header:
- Logo text: Vortiq Dynamics
- Navigation: Home | Services | Solutions | Case Studies | About | Careers | Contact
- Highlighted CTA button: Request Quote

The homepage hero section must include:
- Headline: Tech Solutions for Connected, Intelligent Businesses
- Subtext: IoT, cloud, web, mobile, embedded systems, and edge AI solutions built for startups, businesses, and industries.
- Buttons: Request Quote and Explore Services
- An abstract PCB/cloud animation with moving circuit traces, cloud nodes, and optional dashboard/device preview.
- A typewriter-style text/writing effect in the hero.

Design requirements:
- Straightforward, trustworthy, and professional
- Strong brand color with a high-tech feel
- Clean service icons
- Realistic technology imagery or polished visual placeholders
- Sharp layout, strong spacing, and clear hierarchy
- Mobile responsive
- Fast-loading static pages
- Accessible contrast and readable typography
- Premium but not flashy
- Smooth lightweight animation while scrolling
- Reduced-motion fallback

Use this validated palette:
- Deep Teal: #005461, primary backgrounds, header, footer, primary elements
- Dark Teal: #1E3D40, sections, cards, hover states
- Mint Green: #A8E6CF, highlights, icons, subtle accents
- Bright Green: #3ED16F, buttons, CTAs, active states
- Soft Yellow: #FFF57E, badges and important accents
- Soft Pink: #FF7EB6, decorative micro-accents and diagrams only
- Warm Cream: #FBF5DD, light section backgrounds and cards
- Slate Gray: #7A8C8E, devices, icons, secondary elements
- Deep Charcoal: #0B1F23, text, dark surfaces, high contrast areas

Palette rules:
- Use white text on deep teal, dark teal, and deep charcoal.
- Use deep charcoal text on bright green CTA buttons.
- Do not use white text on bright green CTA buttons.
- Use yellow and pink sparingly so the brand stays premium and technical.

Animation requirements:
- Use abstract PCB and cloud animation in the hero.
- Add scroll-triggered reveals for sections and cards.
- Add a capability-flow section where hardware, firmware, IoT, cloud, dashboard, mobile app, and edge AI layers animate into connection.
- Include subtle motion concepts for electronics/PCB boards, Docker/Kubernetes-style cloud integration, cloud nodes, mobile UI panels, and edge AI.
- Keep animations lightweight with CSS, SVG, Intersection Observer, or minimal JavaScript.
- Do not use heavy animation libraries unless absolutely necessary.

Services catalog section title:
What We Build

Service cards:
1. IoT Solutions
   Sensors, device connectivity, monitoring, automation
2. Cloud Solutions
   Cloud infrastructure, APIs, dashboards, scalable backend systems
3. Web Applications
   Admin panels, business platforms, portals, SaaS tools
4. Mobile Applications
   Android/iOS apps, cross-platform apps, field operation apps
5. Websites
   Business websites, product websites, landing pages
6. Embedded Systems
   Firmware, microcontrollers, device integration
7. Edge AI
   On-device intelligence, vision AI, predictive systems

All seven service cards must be equally visible. Do not hide services behind tabs, carousels, or collapsed content on desktop.

Add an animated capability flow:
Hardware -> Firmware -> IoT -> Cloud -> Dashboard -> Mobile App -> Edge AI

Add a process section titled "How We Work" with these steps:
1. Understand Requirement
2. Design Architecture
3. Build Prototype
4. Develop and Integrate
5. Test, Deploy, Support

Add a tech stack section with these categories:
- Frontend: React, Next.js, HTML, CSS
- Backend: Node.js, Python, APIs
- Cloud: AWS, Azure, Google Cloud, Cloudflare
- IoT/Embedded: RTOS, IoT, hardware and PCB designing, Firmware
- Mobile: Flutter, React Native, Android
- AI/Edge AI: Edge AI products and services, Autonomous Navigation, ADAS

Add a case studies page and homepage preview with these cards:
1. Industrial IoT Monitoring System
2. Cloud Dashboard for Business Operations
3. Mobile App for Field Teams

Each case study card must include:
- Problem
- Solution
- Result

About company section:
Vortiq Dynamics helps businesses transform ideas into reliable technology systems, from connected devices to cloud platforms and intelligent applications.

Key points:
- End-to-end development
- Engineering-first approach
- Scalable architecture
- Support from concept to deployment

Create a Contact / Request Quote form with:
- Name
- Company
- Email
- Phone
- Service Interested In
- Project Brief
- Budget Range
- Submit Button

Create a Careers form with:
- Name
- Email
- Phone
- Role Interested In
- Experience
- Resume Upload
- Message
- Submit Button

FAQ questions:
- What type of projects do you handle?
- Do you work with startups?
- Can you build both hardware and software?
- Do you provide maintenance after delivery?
- Can you develop custom IoT and embedded systems?

Footer must include:
- Logo
- Short tagline
- Quick Links
- Services
- Contact details
- Social links
- Copyright

Media and admin-readiness:
- Use placeholder images for now.
- Build image components so real work images can replace placeholders later.
- Add an ambient image treatment where the main image is clear and the same image appears behind it enlarged, blurred, and softened to create a contrast-matching background.
- Structure content in reusable data objects or JSON-like arrays where practical, so a future admin panel can provide content without redesigning the site.
- Do not build the admin panel yet, but make the website capable of accepting future admin-managed data for hero, services, case studies, gallery/work images, FAQ, careers, contact, and footer content.

The final website should make Vortiq Dynamics feel like a serious technology partner that can take an idea from concept through engineering, deployment, and support.
```
